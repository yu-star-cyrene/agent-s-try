# 公开仓库加密工作流

## 每次推进后的固定收尾动作

以后每次编辑代码、更新文档、调整规则、运行评测、修复脚本或推进项目阶段后，都必须把推进后的当前状态写入：

```text
C:\Users\yu\Desktop\大创\进度记录表.txt
```

进度记录至少应包含：

- 本次推进目标
- 实际修改或新增的内容
- 当前项目状态
- 已执行的验证命令和结果
- 若涉及受保护源码或数据，是否已重新生成 `private_bundle.zip.dcb`
- 当前仍需注意的问题
- 下一步建议

如果本次改动涉及 `src/`、`scripts/`、`tests/`、`configs/`、`data/rules/`、`data/samples/` 或真实图片等受保护内容，收尾时还应重新运行加密流程，确保公开仓库里的 `private_bundle.zip.dcb` 对应最新状态。若要发布到 GitHub，优先使用：

```bat
post_github.bat "update: 本次进度说明"
```

这条规则的目的很简单：让项目每一步都有记录、有验证、有可恢复状态，协作人也能清楚知道当前项目推进到哪里。

## 目标

项目仓库需要公开，但重点源码、真实规则、真实样本和后续真实图片不应直接公开。因此当前采用“公开说明 + 加密包”的方式：

- 公开仓库保留 README、docs、materials 说明、加密工具和加密包。
- 重点源码和真实数据打包进 `private_bundle.zip.dcb`。
- 继续开发前必须用共享 key 解密。
- 上传前由 `post_github.bat` 自动加密并移除明文。

## 受保护内容

受保护路径写在：

```text
.private-bundle-manifest.json
```

当前包括：

- `src/`
- `scripts/`
- `tests/`
- `configs/`
- `data/rules/`
- `data/samples/`
- `data/raw/images/`
- `data/raw/uploads/`
- `run_demo.py`
- `run_web.py`
- `run_web.bat`

## 继续开发

clone 或 pull 公开仓库后，先执行：

```bat
tools\decrypt_private.bat
```

输入共享 key 后，脚本会恢复受保护内容。

也可以在当前终端临时设置 key：

```bat
set DACHUANG_PRIVATE_KEY=你的共享key
tools\decrypt_private.bat
```

不要把 key 写入仓库。

## 最简单协作流程

如果你是协作人，完整流程只需要记住这四步：

```bat
git clone 仓库地址
cd 大创多模态
tools\decrypt_private.bat
```

输入共享 key 后开始开发。开发完成后：

```bat
post_github.bat "update: 本次进度说明"
```

再次输入同一个共享 key。脚本会自动加密、删除受保护明文、提交并推送。

也就是说，真实协作流程就是：

```text
clone/pull
  ↓
key 解密
  ↓
推进项目
  ↓
key 加密
  ↓
post 到 GitHub
```

## 上传新进度

正常开发完成后运行：

```bat
post_github.bat "update: 你的进度说明"
```

这个脚本会自动执行：

1. 调用 `tools\encrypt_private.bat --remove-plaintext`
2. 输入共享 key
3. 生成或更新 `private_bundle.zip.dcb`
4. 删除受保护明文路径
5. 执行 `git add .`
6. 创建 commit
7. push 到 GitHub

如果只是本地调试，不想上传，可以手动运行：

```bat
tools\encrypt_private.bat
```

这会生成加密包但保留明文。真正公开上传前应使用 `--remove-plaintext`。

## 临时跳过加密

一般不要跳过。若只提交公开文档且确认没有明文敏感内容，可以在当前终端设置：

```bat
set SKIP_PRIVATE_ENCRYPT=1
post_github.bat "docs: update public notes"
```

## 安全注意

- key 不要发到 GitHub、聊天公开频道或文档里。
- key 泄露后，需要换 key 并重新生成 `private_bundle.zip.dcb`。
- `private_bundle.zip.dcb` 不是给陌生人公开解密的，只是防止公开仓库直接暴露源码和真实数据。
- 当前加密实现为 `PBKDF2 + AES-256-CBC + HMAC-SHA256`，不依赖外部安装工具。

## 防呆校验

当前脚本已经加入以下校验，避免生成或恢复空包：

- 加密前会检查临时 payload 中是否真的包含受保护文件。
- 加密包内会写入 `_private_bundle_manifest.json` 和内部统计信息。
- 解密时会检查包内是否存在受保护文件；如果是空包会直接报错，不会再恢复空目录。
- 解密失败也会自动清理临时目录。
- 加密包会过滤 `__pycache__/`、`.pyc`、`.pyo`、`.pytest_cache/`，避免把 Python 缓存塞进包里。
