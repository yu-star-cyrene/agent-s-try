# 项目目录分类

## 顶层目录说明

| 目录 | 作用 |
| --- | --- |
| `materials/` | 存放脱敏说明和公开参考资料占位；原始申报材料单独保存在仓库外 |
| `docs/` | 存放项目现状、架构说明、分工建议、阶段方案 |
| `configs/` | 模型配置、运行参数、接口示例配置 |
| `data/raw/` | 原始图像、原始文本、原始举证数据；当前图片仍以占位路径为主 |
| `data/interim/` | 中间处理产物，如 OCR 文本、临时 JSON、图像描述结果 |
| `data/processed/` | 清洗后的训练、评测或展示数据 |
| `data/rules/` | 结构化规则库和规则 Schema |
| `data/samples/` | 样本集、样本 Schema、单条 demo 请求 |
| `src/` | 项目主代码 |
| `tests/` | 冒烟测试与后续单元测试 |
| `outputs/` | 本地运行日志、评测报告、导出结果 |
| `examples/` | 后续可放演示输入输出和典型案例 |
| `scripts/` | 批量评测脚本和后续数据处理脚本 |

## `src/` 下的模块划分

| 目录 | 作用 |
| --- | --- |
| `src/core/` | 公共数据结构、基础类型定义 |
| `src/modules/rule_engine/` | 规则加载、启用规则筛选、规则优先级排序 |
| `src/modules/vision_precheck/` | 代码辅助视觉预检，提取图片状态、尺寸、EXIF 等证据 |
| `src/modules/model_gateway/` | 模型接口封装；当前包含 `MockModelClient`、`QwenVLClient` 和模型工厂 |
| `src/modules/prompt_engine/` | 规则驱动 Prompt 构建 |
| `src/modules/postprocess/` | 模型输出解析、字段修补、`rule_hits` 提取 |
| `src/modules/evaluation/` | 字段完整率等评测指标 |
| `src/app/` | Web 和 CLI 演示入口 |

| `scripts/evaluate_samples.py` | 批量评测入口 |

| `outputs/reports/evaluation_details.json` | 本地生成的逐样本评测详情 |
| `outputs/reports/evaluation_details.csv` | 适合 Excel/WPS 打开的评测表格 |

## 推荐使用方式

### 规则与样本线

- 优先维护 `data/rules/` 和 `data/samples/`
- 每次新增规则后，要补对应样本和标准答案

### 推理与评测线

- 优先维护 `src/modules/vision_precheck/`、`src/modules/model_gateway/`、`src/modules/prompt_engine/`、`src/modules/postprocess/`、`scripts/`
- 每次修改视觉预检、Prompt、Parser、Mock 或真实模型接口后，都运行批量评测

### 展示与汇报线

- 优先维护 `src/app/`、`outputs/reports/`、`docs/`
- 用评测报告、Web 页面和典型样本形成中期检查或答辩材料
