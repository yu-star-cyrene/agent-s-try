# 项目目录分类

## 顶层目录说明

| 目录 | 作用 |
| --- | --- |
| `materials/` | 存放脱敏说明和公开参考资料占位；原始申报材料单独保存在仓库外 |
| `docs/` | 存放项目现状、架构说明、模型方案、分工建议、阶段方案 |
| `configs/` | 模型配置、运行参数、接口示例配置 |
| `data/raw/` | 原始图像、上传图片、原始举证数据；`uploads/` 用于 Web/API 接收图片 |
| `data/interim/` | 中间处理产物，如 OCR 文本、临时 JSON、图像描述结果 |
| `data/processed/` | 清洗后的训练、评测或展示数据 |
| `data/rules/` | 结构化规则库和规则 Schema |
| `data/samples/` | 样本集、样本 Schema、单条 demo 请求 |
| `src/` | 项目主代码 |
| `tests/` | 冒烟测试与后续单元测试 |
| `outputs/` | 本地运行日志、评测报告、导出结果；默认不提交 |
| `examples/` | 后续可放演示输入输出和典型案例 |
| `scripts/` | 批量评测脚本和后续数据处理脚本 |
| `tools/` | 加密、解密等协作工具 |

## `src/` 下的模块划分

| 目录 | 作用 |
| --- | --- |
| `src/core/` | 公共数据结构、基础类型定义 |
| `src/modules/rule_engine/` | 规则加载、启用规则筛选、规则优先级排序 |
| `src/modules/vision_precheck/` | 代码辅助视觉预检，提取图片状态、尺寸、EXIF 等证据 |
| `src/modules/model_gateway/` | 模型接口封装；当前包含 Mock、Qwen-VL API、本地 Qwen2.5-VL 和模型工厂 |
| `src/modules/prompt_engine/` | 规则驱动 Prompt 构建，并注入 `parcel_context` 约束 |
| `src/modules/postprocess/` | 模型输出解析、字段修补、`rule_hits` 提取 |
| `src/modules/evaluation/` | 字段完整率等评测指标 |
| `src/app/` | Web 演示入口和通用模型审查 API |

## API 相关路径

当前 Web 服务由 `python run_web.py` 启动，默认监听：

```text
http://127.0.0.1:5000
```

可用接口：

| 接口 | 方法 | 作用 |
| --- | --- | --- |
| `/` | GET/POST | 本地网页演示，支持上传图片和填写图斑上下文 |
| `/api/v1/health` | GET | 服务健康检查 |
| `/api/v1/inspect` | POST | 推荐模型审查接口 |
| `/api/inspect` | POST | 兼容短路径 |

推荐由其他前端传入：

- `image_file` 或 `image_base64`
- `parcel_id`
- `parcel_context`
- `text_description`
- `rules`
- `metadata`

## 受保护路径说明

当前以下路径不直接明文提交到公开仓库，而是进入 `private_bundle.zip.dcb`：

- `src/`
- `scripts/`
- `tests/`
- `configs/`
- `data/rules/`
- `data/samples/`
- `data/raw/images/`
- `data/raw/uploads/`
- `run_demo.py`
- `run_web.py`
- `run_web.bat`

所以普通 `git status` 可能只显示 README 或 docs 变化，但实际上受保护源码已经改动。发布前必须运行：

```bat
post_github.bat "update: 本次进度说明"
```

## 推荐使用方式

### 规则与样本线

- 优先维护 `data/rules/` 和 `data/samples/`
- 每次新增规则后，要补对应样本和标准答案
- 后续应把 `parcel_context` 结构纳入样本 Schema

### 地图上下文与接口线

- 优先维护 `src/app/web_app.py`、`src/app/templates/index.html`
- 保持 `/api/v1/inspect` 向前兼容
- 其他前端只通过 API 调用模型服务，不直接调用模型代码

### 推理与评测线

- 优先维护 `src/modules/vision_precheck/`、`src/modules/model_gateway/`、`src/modules/prompt_engine/`、`src/modules/postprocess/`、`scripts/`
- 每次修改视觉预检、Prompt、Parser、Mock 或真实模型接口后，都运行测试和批量评测

### 展示与汇报线

- 优先维护 `src/app/`、`outputs/reports/`、`docs/`
- 用 Web 页面、API 调用示例、评测报告和典型样本形成中期检查或答辩材料
