# 初步工作分工

这份分工按 5 人小组来设计，和你们现有材料最匹配。现在先按“角色”分，不强行绑定具体姓名，后面你们可以再按擅长方向替换。

## 分工建议

| 角色 | 负责模块 | 主要任务 | 主要目录 | 核心产出 |
| --- | --- | --- | --- | --- |
| 角色 A | 业务规则与样本 | 整理图斑审查规则、异常场景、输出字段、标准答案样例 | `docs/` `data/rules/` `data/samples/` | 规则文档、规则 JSON、样本库 |
| 角色 B | 模型接入与工程底座 | 选模型、封装接口、处理图片/文本输入、记录日志 | `configs/` `src/modules/model_gateway/` | 模型接口、调用配置、日志方案 |
| 角色 C | Prompt 与推理设计 | 设计系统提示词、Few-shot 模板、审查步骤与约束 | `src/modules/prompt_engine/` `data/samples/` | Prompt 模板、样例提示词 |
| 角色 D | 结果结构化与评测 | 解析模型输出、统一字段、做完整率和准确率统计 | `src/modules/postprocess/` `src/modules/evaluation/` `tests/` | 解析器、评测脚本、结果报告 |
| 角色 E | 集成与演示 | 串联全流程、搭 demo、整理答辩展示材料 | `src/main.py` `src/app/` `outputs/` `docs/` | 可运行 demo、流程图、汇报材料 |

## 协作顺序

1. 角色 A 先把规则、字段、样本定义出来。
2. 角色 B 和角色 C 并行推进，一个做模型接口，一个做提示词模板。
3. 角色 D 在接口与提示词初版出来后，接结果结构化和评测。
4. 角色 E 最后把全链路串起来，做展示和答辩版本。

## 每个人最少要交的东西

- 角色 A：一份规则文档 + 一份规则 JSON + 一批样本
- 角色 B：一个可调用的模型接口模块
- 角色 C：一版稳定的 Prompt 模板
- 角色 D：一个结果解析器 + 一份评测脚本
- 角色 E：一个能跑的 demo + 一份展示说明

## 当前阶段最适合先做什么

如果你们现在还在“立项和准备”阶段，建议优先级如下：

1. 先由角色 A 完成规则和样本定义
2. 再由角色 B 做接口骨架
3. 再由角色 C 写第一版提示词
4. 然后角色 D 接结构化输出
5. 最后角色 E 做集成和展示

这样推进最稳，也最容易在中期检查时拿出像样成果。
