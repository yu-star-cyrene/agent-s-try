# 当前工作分工与推进建议

这份分工基于当前项目进展更新。项目已经完成规则库、Prompt 链路、结构化输出、26 条规则覆盖样本、批量评测闭环、代码辅助视觉预检层和 Qwen-VL 接入骨架，因此后续分工应转向“验证真实模型、补真实图像、增强视觉证据、做规则级评测、准备展示”。

## 当前模块状态

| 角色 | 负责模块 | 当前状态 | 下一步重点 |
| --- | --- | --- | --- |
| 角色 A | 业务规则与样本 | 已有结构化规则库和 26 条规则覆盖样本 | 补真实或脱敏图片，复核标准答案 |
| 角色 B | 模型接入与工程底座 | 已有 `MockModelClient` 和 `QwenVLClient` 骨架 | 设置 API key，跑通真实 Qwen-VL 调用 |
| 角色 C | Prompt 与推理设计 | 已支持结构化规则和视觉预检证据注入 Prompt | 基于真实模型失败样本优化 Prompt |
| 角色 D | 结果结构化与评测 | 已有 Parser、`rule_hits`、批量评测和规则级指标 | 增强 Schema 校验，分析真实模型规则级结果 |
| 角色 E | 集成与演示 | Web/CLI 已可运行 | 把评测结果做成展示图表和汇报材料 |

## 推荐协作顺序

1. 角色 B 先设置 Qwen-VL API key，用 `configs/model.example.json` 跑通一条真实调用。
2. 角色 A 为 26 条样本补真实或脱敏图片，并复核标准答案。
3. 角色 D 继续运行 `scripts/evaluate_samples.py`，对比 Mock 与真实模型的规则级指标。
4. 角色 C 根据真实模型失败样本优化 Prompt 模板。
5. 角色 E 整理 Web 截图、评测表格、系统流程图和答辩讲解。

## 每个角色下一阶段最少交付物

- 角色 A：真实或脱敏图片、规则覆盖清单、标准答案复核说明。
- 角色 B：真实模型调用验证记录、配置说明、失败重试方案。
- 角色 C：一版稳定 Prompt 模板、若干失败样本优化记录。
- 角色 D：评测 summary、details、CSV 报告，以及字段校验方案。
- 角色 E：演示流程、页面截图、PPT 中可用的指标表。

## 当前优先级

| 优先级 | 事项 | 说明 |
| --- | --- | --- |
| 1 | 跑通 Qwen-VL 真实调用 | 当前已有客户端骨架，但还需要真实 key 验证 |
| 2 | 补真实或脱敏图片 | 当前 26 条样本仍以占位图片路径为主 |
| 3 | 保持批量评测可运行 | 每次改规则、Prompt、模型后都要重新跑评测 |
| 4 | 完善视觉预检与规则级指标 | 补 EXIF/GPS、OpenCV、YOLO/SAM 等可解释证据 |
| 5 | 准备答辩展示材料 | 用评测报告和典型案例支撑项目成果 |
