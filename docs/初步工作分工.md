# 当前工作分工与推进建议

这份分工基于当前最新进展更新。项目已经完成规则库、Prompt 链路、结构化输出、26 条规则覆盖样本、批量评测闭环、代码辅助视觉预检、Qwen-VL API 接入骨架、本地 Qwen2.5-VL 入口、Web 上传页面和通用模型审查 API。

当前最重要的架构调整是：系统从“手填地类 + 上传图片”的演示模式，升级为“地图图斑上下文 `parcel_context` + 上传图片核查”的模型服务模式。后续分工也应围绕这一点展开。

## 当前模块状态

| 角色 | 负责模块 | 当前状态 | 下一步重点 |
| --- | --- | --- | --- |
| 角色 A | 业务规则与样本 | 已有 R001-R006 结构化规则和 26 条规则覆盖样本 | 补真实/脱敏图片，复核标准答案，补 `parcel_context` 字段 |
| 角色 B | GIS/地图上下文 | 已明确 `parcel_context` 结构，但尚未接真实地图数据 | 准备图斑边界、图层来源、要素 ID、地类来源、种植人/责任主体字段 |
| 角色 C | 模型服务与 API | 已有 `/api/v1/inspect`，支持图片上传、`image_base64`、`parcel_context` | 保持接口稳定，补接口示例，供其他前端调用 |
| 角色 D | 模型接入与 Prompt | 已有 Mock、Qwen-VL API、本地 Qwen2.5-VL；Prompt 已约束模型不得凭图片确认地类/权属 | 用真实图片测试 API 输出，根据失败样本优化 Prompt |
| 角色 E | 评测与结果分析 | 已有 Parser、`rule_hits`、批量评测和规则级指标 | 对比 Mock 与真实模型，输出失败样本分析 |
| 角色 F | 展示与答辩材料 | Web 页面已能展示结构化结果 | 整理 Web 截图、API 调用流程、评测图表和典型案例 |

## 推荐协作顺序

1. 角色 B 先整理一批模拟或真实脱敏的地图图斑上下文，至少包含 `authoritative_land_type`、`map_layer`、`map_feature_id`、`spatial_reference`。
2. 角色 A 为这些图斑准备 3-6 张真实或脱敏图片，并标注标准答案。
3. 角色 C 使用 `/api/v1/inspect` 跑通前端上传图片到模型服务的流程。
4. 角色 D 使用 Qwen-VL API 跑真实模型输出，检查 `reason`、`evidence` 和 `rule_hits` 是否符合业务表达。
5. 角色 E 用同一批样本对比 Mock 与真实模型，记录规则级 recall/precision 和失败原因。
6. 角色 F 整理 Web 页面、API 返回 JSON、评测结果和系统流程图，用于中期检查或答辩。

## 每个角色下一阶段最少交付物

- 角色 A：真实或脱敏图片、规则覆盖清单、标准答案复核说明。
- 角色 B：`parcel_context` 示例表，包括图层、图斑要素 ID、权威地类、地类来源、位置依据、种植人/责任主体。
- 角色 C：API 调用示例，包括 multipart 上传和 JSON/base64 两种方式。
- 角色 D：真实模型调用记录、Prompt 优化记录、失败样本修正建议。
- 角色 E：评测 summary、details、CSV 报告，以及规则级失败样本分析。
- 角色 F：演示流程、页面截图、PPT 中可用的指标表和系统架构图。

## 当前优先级

| 优先级 | 事项 | 说明 |
| --- | --- | --- |
| 1 | 用 Web/API 上传真实或脱敏图片 | 当前接口已可用，需要验证真实视觉效果 |
| 2 | 补 `parcel_context` 小样本 | 地图图斑上下文是正式业务系统接入关键 |
| 3 | 跑通 3-6 条真实模型小评测 | 不要先跑全量 26 条，先控成本看质量 |
| 4 | 根据失败样本优化 Prompt | 重点看模型是否正确区分“权威上下文”和“图片现状” |
| 5 | 发布加密包 | 受保护源码改动后必须重新生成 `private_bundle.zip.dcb` |
| 6 | 准备答辩展示材料 | 用 API 流程、Web 截图和评测结果支撑项目成果 |

## 当前可直接运行的命令

开发前恢复受保护源码：

```bat
tools\decrypt_private.bat
```

启动 Web/API 服务：

```bat
set DACHUANG_MODEL_CONFIG=configs\model.example.json
python run_web.py
```

运行单条 Demo：

```bat
python run_demo.py
```

运行批量评测：

```bat
python scripts\evaluate_samples.py
```

运行测试：

```bat
python -m unittest discover -s tests -t . -p "test*.py"
```

发布当前进度：

```bat
post_github.bat "update: 本次进度说明"
```
