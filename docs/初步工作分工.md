# 当前工作分工与推进建议

这份分工基于当前项目进展更新。项目已经完成规则库、Prompt 链路、结构化输出和第一版批量评测闭环，因此后续分工应从“搭骨架”转向“扩数据、接模型、做评测、准备展示”。

## 当前模块状态

| 角色 | 负责模块 | 当前状态 | 下一步重点 |
| --- | --- | --- | --- |
| 角色 A | 业务规则与样本 | 已有结构化规则库和 5 条 MVP 样本 | 扩充规则场景，补样本到 20-30 条 |
| 角色 B | 模型接入与工程底座 | 当前使用 `MockModelClient` | 选择并接入真实多模态模型 |
| 角色 C | Prompt 与推理设计 | 已支持结构化规则注入 Prompt | 基于评测失败样本优化 Prompt |
| 角色 D | 结果结构化与评测 | 已有 Parser、`rule_hits` 和批量评测脚本 | 增强 Schema 校验，完善评测指标 |
| 角色 E | 集成与演示 | Web/CLI 已可运行 | 把评测结果做成展示图表和汇报材料 |

## 推荐协作顺序

1. 角色 A 先扩充 `sample_set.json`，保证每条规则至少有 2-3 条样本。
2. 角色 D 继续运行 `scripts/evaluate_samples.py`，检查新增样本是否能形成稳定指标。
3. 角色 C 根据评测失败样本优化 Prompt 模板。
4. 角色 B 接入真实多模态模型，并沿用同一套样本重新评测。
5. 角色 E 整理 Web 截图、评测表格、系统流程图和答辩讲解。

## 每个角色下一阶段最少交付物

- 角色 A：20-30 条样本、规则覆盖清单、标准答案说明。
- 角色 B：一个真实模型调用客户端、配置说明、失败重试方案。
- 角色 C：一版稳定 Prompt 模板、若干失败样本优化记录。
- 角色 D：评测 summary、details、CSV 报告，以及字段校验方案。
- 角色 E：演示流程、页面截图、PPT 中可用的指标表。

## 当前优先级

| 优先级 | 事项 | 说明 |
| --- | --- | --- |
| 1 | 扩充样本集 | 当前只有 5 条，仍是 MVP 级别 |
| 2 | 保持批量评测可运行 | 每次改规则、Prompt、模型后都要重新跑评测 |
| 3 | 接入真实多模态模型 | 用同一批样本对比 Mock 和真实模型 |
| 4 | 完善 Schema 校验 | 防止规则和样本字段走形 |
| 5 | 准备答辩展示材料 | 用评测报告和典型案例支撑项目成果 |
