# 第一步：规则库与数据格式落地方案

## 为什么第一步先做这个

根据《项目总体判断.pdf》，当前项目已经有主流程骨架，但核心缺口不在“能不能跑”，而在“按什么规则判、用什么格式传、怎么给标准答案”。因此第一步必须先把下面三件事落地：

1. 规则库从占位描述升级为结构化规则。
2. 样本从单条演示升级为可评测任务集。
3. 输入输出字段从口头约定升级为可校验格式。

这一步完成后，后面的 Prompt 设计、模型接入、结果解析、评测统计才会有统一口径。

## 本阶段目标

- 形成规则分析总表的字段标准。
- 完成 `data/rules/rule_template.json` 第一版正式结构。
- 完成 `data/samples/sample_set.json` 第一批样本任务。
- 提供 `rule_schema.json` 与 `sample_schema.json` 作为格式规范底稿。

## 建议使用的专业工具

### 规则分析

- `Excel` 或 `WPS 表格`
- 用途：整理规则分析总表、管理字段、做版本对比、多人协作复核

### 图斑与属性核查

- `QGIS`
- 用途：检查图斑编号、地类、坐标、边界、属性表字段是否合理，辅助设计元数据结构

### 样本标注与复核

- `Label Studio`
- 用途：管理图像、文本、标签、规则命中情况，支持多人复核和统一导出

### 格式校验

- `JSON Schema`
- `Pydantic`
- 用途：锁定字段名、字段类型、必填项，避免后续各模块各写各的

## 规则分析总表建议字段

建议在 `Excel/WPS` 中建立规则分析总表，列名如下：

| 字段名 | 含义 |
| --- | --- |
| 规则ID | 规则唯一编号，如 `R002` |
| 规则名称 | 规则标题 |
| 规则类别 | 如图文一致性、土地利用冲突、举证质量、真实性风险 |
| 适用地类 | 耕地、林地、建设用地、全部地类等 |
| 触发条件 | 能导致规则命中的观察条件 |
| 所需证据 | 图像、文本、元数据等 |
| 风险等级 | `low` / `medium` / `high` |
| 判断逻辑 | 这条规则如何转成结论 |
| 输出结论 | 对应 `issue_type` 等输出值 |
| 是否人工复核 | 是否必须转人工 |
| 优先级 | 0-100，解决规则冲突 |
| 来源 | 项目文档、老师建议、业务总结 |
| 备注 | 特殊说明 |

## 规则 JSON 结构约定

每条规则至少包含以下字段：

- `id`
- `title`
- `category`
- `severity`
- `applies_to`
- `description`
- `trigger_conditions`
- `required_evidence`
- `decision_logic`
- `output`
- `priority`
- `enabled`
- `source`

其中：

- `description` 继续保留，兼容现有 `rule_loader.py`
- `output` 用于给后续 Prompt 和 Parser 提供统一目标
- `priority` 用于解决多规则同时命中的冲突问题

## 样本任务格式约定

每条样本至少包含：

- `sample_id`
- `parcel_id`
- `image_path`
- `text_description`
- `land_type`
- `metadata`
- `expected_result`
- `applied_rule_ids`

其中 `expected_result` 是标准答案，不只是模型输出示例，而是评测基准。

## 当前第一批必须覆盖的场景

第一轮样本建议优先覆盖：

1. 耕地疑似建筑占用
2. 图文一致但证据不足
3. 完全正常样本
4. 举证真实性风险
5. 林地疑似占用

## 下一步如何接到代码

第一步完成后，建议按这个顺序接入代码：

1. `prompt_builder.py` 读取扩展后的规则字段，生成更稳的 Prompt。
2. `parser.py` 按 `sample_schema.json` 和目标输出字段补全结构化解析。
3. `src/main.py` 支持批量读取 `sample_set.json` 做评测。
4. `tests/` 增加规则格式和样本格式校验测试。

## 本阶段完成定义

当满足以下条件时，可认为“项目第一步完成”：

- 规则总表字段固定
- 规则 JSON 可供程序读取
- 样本集可供模型测试
- 字段格式有 Schema 可校验
- 团队对“异常、疑似异常、人工复核”的口径一致
