# 材料梳理与当前架构

## 项目定位

本项目不是泛化聊天机器人，而是面向国土图斑审查场景的垂直预检系统。系统目标是融合地块图像、文本说明、元数据和业务规则，先做一轮机器预审，提前筛出明显异常、疑似异常和需要人工复核的样本。

## 现实痛点

- 外业举证强依赖人工，效率低。
- 福建山地丘陵地形复杂，图斑碎片化、隐蔽化明显。
- 存在异地拍照、旧图新用、死角漏拍、避重就轻等造假风险。
- 单靠人工比对，难以同时处理图像内容、空间线索和业务规则。

## 当前已完成内容

### 1. 规则库结构化

原来的规则只有 `id`、`title`、`description`，现在已经升级为结构化规则，包含：

- 规则类别
- 风险等级
- 适用地类
- 触发条件
- 所需证据
- 判断逻辑
- 输出口径
- 优先级
- 是否启用

对应文件：

- `data/rules/rule_template.json`
- `data/rules/rule_schema.json`

### 2. 样本集与标准答案

当前已经建立第二版规则覆盖样本集，样本数量从 5 条扩充到 26 条。每条样本包含输入信息和标准答案：

- 图斑编号
- 图片路径
- 地类信息
- 文本说明
- 元数据
- 预期结果
- 命中规则 ID

对应文件：

- `data/samples/sample_set.json`
- `data/samples/sample_schema.json`

### 3. Prompt 推理链路

`prompt_builder.py` 已经可以读取结构化规则字段，把规则 ID、触发条件、证据要求、判断逻辑和 JSON 输出约束放进 Prompt。

这意味着模型不再只是“看图说话”，而是按规则库进行审查。

### 4. 代码辅助视觉预检

项目已新增 `src/modules/vision_precheck/`，在调用多模态模型前先做低成本、可解释的图片证据提取。

当前已支持：

- 图片路径和文件存在性检查
- 图片可读性检查
- 图片尺寸和长宽比提取
- EXIF 时间和 GPS 信息提取
- 将结果写入 `metadata.vision_precheck`

这一步的意义是：不要把全部图像判断都交给大模型，而是先用代码提取稳定证据，再让多模态模型结合规则进行综合判断。

### 5. Qwen-VL 模型接入骨架

项目已新增 `QwenVLClient` 和模型工厂：

- `src/modules/model_gateway/qwen_vl_client.py`
- `src/modules/model_gateway/factory.py`

默认仍使用 `MockModelClient`，但可以通过 `DACHUANG_MODEL_CONFIG` 指向配置文件，切换到 Qwen-VL API 调用。

### 6. 结构化输出与规则追溯

输出结果已经支持：

- `is_abnormal`
- `issue_type`
- `reason`
- `confidence`
- `requires_manual_review`
- `evidence`
- `rule_hits`

其中 `rule_hits` 用于追溯命中了哪些业务规则，是后续答辩和评测的重要依据。

### 7. 批量评测闭环

当前已经新增批量评测脚本：

```bash
python scripts/evaluate_samples.py
```

运行后会生成：

- `outputs/reports/evaluation_summary.json`
- `outputs/reports/evaluation_details.json`
- `outputs/reports/evaluation_details.csv`

其中 `evaluation_summary.json` 已包含总体准确率和 `rule_metrics`，可以按 R001-R006 查看每条规则的样本覆盖、召回率和精确率。

这一步让项目从“单条 Demo”升级为“有样本、有标准答案、有指标报告”的可评测系统。

## 当前系统流程

```text
样本输入
    ↓
代码辅助视觉预检
    ↓
加载结构化规则库
    ↓
构建规则驱动 Prompt
    ↓
模型推理（默认 Mock，可配置 Qwen-VL）
    ↓
解析结构化输出
    ↓
生成 rule_hits 与人工复核建议
    ↓
批量评测并输出报告
```

## 当前限制

- 当前模型仍是 `MockModelClient`，主要用于验证流程，不代表真实模型效果。
- Qwen-VL 客户端骨架已经完成，但真实 API 调用还需要设置 key 后验证。
- 当前样本已覆盖 R001-R006，但图片路径仍是占位路径，尚未接入真实图像分析。
- 当前样本可以支撑规则流程验证，但还需要真实或脱敏图像来支撑最终答辩。
- Schema 文件已经存在，但还没有在主流程中强制校验。

## 后续重点

1. 设置 Qwen-VL API key，跑通一条真实模型调用。
2. 准备真实或脱敏图像样本。
3. 对 Mock 与真实模型在同一套样本上做对比评测。
4. 继续增强视觉预检层，加入 EXIF 时间/GPS 对比、OpenCV 质量检测、YOLO/SAM 证据提取。
5. 将评测结果整理成图表和答辩材料。
