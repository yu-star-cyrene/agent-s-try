# 模型接入与视觉辅助方案

## 当前结论

项目不采用“只让大模型直接看图判断”的路线，而采用：

```text
代码辅助视觉预检
    ↓
多模态模型图文理解
    ↓
规则库约束推理
    ↓
结构化输出与评测
```

这样做的原因是：图斑审查需要可解释证据、稳定字段、规则追溯和低成本批量评测。传统代码适合提取确定性证据，多模态模型适合综合图像、文本、元数据和规则做最终判断。

## 推荐主模型

当前推荐第一优先接入 Qwen-VL 系列，优先使用 API 方式验证：

- `Qwen3-VL`：适合后续视觉理解、空间关系、OCR、目标定位等能力扩展。
- `Qwen2.5-VL`：适合作为成熟可用的视觉语言模型路线。
- API 侧可先用兼容 OpenAI Chat Completions 结构的服务，降低工程接入成本。

推荐原因：

- 中文业务规则理解更贴合项目场景。
- 更适合在大创材料中表达“国产多模态模型接入”。
- 后续可从 API 迁移到 ModelScope 或本地开源权重。
- 当前项目已有 `ModelClient.inspect(prompt, image_path)` 抽象层，替换模型成本低。

## 已完成的工程落地

### 1. 视觉预检层

新增模块：

```text
src/modules/vision_precheck/
```

当前能力：

- 判断图片路径是否为空。
- 判断图片文件是否存在。
- 判断图片文件是否可读取。
- 若本机安装 Pillow，可读取图片尺寸、长宽比和 EXIF 信息。
- 输出稳定 JSON 字段，写入 `metadata.vision_precheck`。

当前输出示例：

```json
{
  "image_exists": false,
  "image_readable": false,
  "signals": ["image_file_missing"],
  "quality_flags": []
}
```

注意：当前不会因为样本图片暂缺而阻断流程，只会把图片状态作为辅助证据传给后续 Prompt。

### 2. Qwen-VL 客户端骨架

新增文件：

```text
src/modules/model_gateway/qwen_vl_client.py
src/modules/model_gateway/factory.py
```

当前支持：

- 从配置文件读取模型名、API base、key 环境变量、超时、重试等参数。
- 使用图文消息格式构造请求。
- 本地图片存在时自动转为 base64 data URL。
- 未设置 API key 时，在联网前明确报错。

### 3. 配置化 pipeline

新增入口：

```python
build_pipeline_from_config(config_path)
```

默认仍使用 `MockModelClient`，保证本地测试和演示稳定。

如果要启用真实模型，可设置：

```bat
set DACHUANG_MODEL_CONFIG=configs\model.example.json
set DASHSCOPE_API_KEY=你的DashScopeKey
```

然后运行：

```bat
python run_demo.py
```

或：

```bat
python run_web.py
```

## 后续扩展路线

### 阶段 A：轻量代码证据

优先完善：

- 图片尺寸、分辨率、长宽比异常。
- EXIF 拍摄时间与任务时间对比。
- EXIF GPS 与图斑位置对比。
- 图片是否过小、不可读、缺少元数据。

### 阶段 B：传统视觉辅助

后续可接：

- OpenCV：模糊、过暗、边缘、纹理、颜色占比等质量与场景线索。
- Rasterio/GDAL/Shapely：遥感图、GeoTIFF、图斑边界、空间裁剪和坐标一致性。
- YOLO：建筑、道路、施工机械、堆料区等目标检测。
- SAM 2：道路、硬化区域、植被区域等分割辅助。

### 阶段 C：多模态模型综合推理

Qwen-VL 负责：

- 看图理解现场内容。
- 结合文本说明和地类信息。
- 参考 `metadata.vision_precheck` 中的代码证据。
- 根据 R001-R006 规则输出合法 JSON。

### 阶段 D：真实模型评测

用同一套 26 条样本进行对比：

- Mock 模型结果
- Qwen-VL 真实模型结果
- 每条规则的 recall / precision
- 失败样本原因分析

## 当前注意事项

- 真实 API key 只能放环境变量，不能写入仓库。
- 当前图片路径仍以占位为主，因此真实模型效果评测还需要后续补图。
- `MockModelClient` 仍然保留，用于无 API key 时跑测试和演示。
- Qwen-VL 客户端骨架已经完成，但真实 API 调用需要设置 `DASHSCOPE_API_KEY` 后再验证。
